# Usage

Use MkDocs to serve the local development site or build the static output from
[site_docs/](../site_docs/) using [mkdocs.yml](../mkdocs.yml).

## Quick start
From the repo root:
```bash
mkdocs serve
```
Open `http://127.0.0.1:8000/`.

## CLI
- Regenerate subject indexes, nav, and the self-test progress manifest:
  ```bash
  source source_me.sh && python generate_pages.py
  ```
- Serve the site locally:
  ```bash
  mkdocs serve
  ```
- Build the static site:
  ```bash
  mkdocs build
  ```

## Examples
- Serve locally for preview:
  ```bash
  mkdocs serve
  ```
- Build output into the git-ignored `site/` directory:
  ```bash
  mkdocs build
  ```

## Inputs and outputs
- Inputs: [site_docs/](../site_docs/) content and [mkdocs.yml](../mkdocs.yml) config.
- Outputs: the git-ignored `site/` directory of static files (generated by `mkdocs build`).
- Generated progress data:
  [site_docs/assets/data/selftest_question_manifest.json](../site_docs/assets/data/selftest_question_manifest.json)
  is produced by [generate_pages.py](../generate_pages.py) from reachable topic
  pages and their self-test includes.

## Self-test progress

The [Self-test Progress](../site_docs/progress/index.md) dashboard tracks which
embedded self-test questions have been answered fully correctly in the current
browser. Progress is local to browser `localStorage` under
`selftest_progress_v1`; it stores completed question IDs and first-correct
timestamps. Wrong answers, attempts, and accuracy are not stored.

Clearing browser site data or using the dashboard reset button removes this
self-test progress. Daily puzzle stats use separate storage keys and are not
affected by the self-test reset.

For the manifest schema, the storage format, and how to add a new self-test
question, see [docs/SELFTEST_PROGRESS.md](SELFTEST_PROGRESS.md).
