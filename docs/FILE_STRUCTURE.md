# File structure

## Top-level layout

- [AGENTS.md](../AGENTS.md): Agent workflow and environment rules.
- [README.md](../README.md): Project overview and quick start.
- [mkdocs.yml](../mkdocs.yml): MkDocs configuration and site navigation.
- [generate_pages.py](../generate_pages.py): Page-generation entry point for
  subject indexes, topic pages, the generated nav block, and the self-test
  manifest.
- [run_bbq_tasks.py](../run_bbq_tasks.py): BBQ batch task runner (imports
  `bioproblems_site`).
- [topics_metadata.yml](../topics_metadata.yml): Single source of truth for subjects
  and topics (titles, descriptions, aliases, links).
- [bioproblems_site/](../bioproblems_site/): Importable generator package for
  metadata, scanning, rendering, nav updates, and self-test manifest creation.
- [site_docs/](../site_docs/): MkDocs content root for pages and assets.
- `site/`: Built site output (generated, git ignored).
- [docs/](.): Repo documentation and standards.
- [bbq_control/](../bbq_control/): BBQ task CSVs, settings, and runner scripts.
- [tools/](../tools/): Helper scripts for content generation tasks.
- [devel/](../devel/): Developer utilities (versioning, changelog tooling).
- [tests/](../tests/): Pytest suite, lint gates, and ASCII compliance checks.

## Key subtrees

- [bioproblems_site/](../bioproblems_site/): Generation package, including
  [bioproblems_site/pipeline.py](../bioproblems_site/pipeline.py),
  [bioproblems_site/metadata.py](../bioproblems_site/metadata.py),
  [bioproblems_site/topic_aliases.py](../bioproblems_site/topic_aliases.py), and
  [bioproblems_site/selftest_manifest.py](../bioproblems_site/selftest_manifest.py).
- [site_docs/](../site_docs/): One folder per subject (for example
  [site_docs/biochemistry/](../site_docs/biochemistry/),
  [site_docs/genetics/](../site_docs/genetics/),
  [site_docs/molecular_biology/](../site_docs/molecular_biology/)), each holding
  `topicNN/index.md` pages.
- [site_docs/assets/](../site_docs/assets/): Shared CSS, JS, fonts, images, and
  generated data.
- [site_docs/assets/scripts/selftest_progress.js](../site_docs/assets/scripts/selftest_progress.js): Browser self-test completion tracking.
- [site_docs/progress/](../site_docs/progress/): Self-test progress dashboard page.
- [site_docs/daily_puzzles/](../site_docs/daily_puzzles/): Daily puzzle pages and
  scripts.
- [site_docs/tutorials/](../site_docs/tutorials/): BBQ and LMS import tutorials.
- [bbq_control/task_files/](../bbq_control/task_files/): Per-subject task CSVs (for
  example `biochem_tasks1.csv`, `biotech_tasks.csv`, `other_tasks.csv`).

## Generated artifacts

Git-ignored outputs and local-only files are listed in [.gitignore](../.gitignore),
including the built `site/` directory, `report_*.txt` lint reports,
`bbq_generation.log*` files, and the `bbq_converter.py` symlink. Subject index
pages and topic pages under [site_docs/](../site_docs/) are regenerated by
[generate_pages.py](../generate_pages.py); subject and topic folders with no
questions are omitted from the rendered nav.

### Topic folder file taxonomy and orphan policy

Each topic folder under [site_docs/](../site_docs/) mixes hand-authored source
with generated output. The reconcile mechanism in
`bioproblems_site/orphan_prune.py` keys every
bbq-derived file, include line, and cache key off the live
`bbq-*-questions.txt` set (the source of truth) and brings the rest back in
lockstep on every gated pipeline run, before the self-test manifest validates.

Per-topic file map:

```
site_docs/<subject>/<topicNN>/
  bbq-<core>-questions.txt         SOURCE OF TRUTH (one per problem variant)
  <name>.pgml | <name>.pg          SOURCE (manual sister master; quarantine)
  index.md                         GENERATED (render_all, bbq-glob driven)
  problem_set_titles.yml           CACHE (keyed by bbq basename; LLM titles)
  downloads/
    selftest-<core>.html           GENERATED from bbq
    human_readable-<core>.html     GENERATED from bbq
    blackboard_qti_v2_1-<core>.zip GENERATED from bbq
    canvas_qti_v1_2-<core>.zip     GENERATED from bbq
    <mapped>.pgml | <mapped>.pg    GENERATED/COPIED into downloads (disposable)
```

Source vs generated split:

- GENERATED: `index.md` content; every file under `downloads/` (the 4 prefixed
  artifacts `selftest-*.html`, `human_readable-*.html`,
  `blackboard_qti_v2_1-*.zip`, `canvas_qti_v1_2-*.zip`, plus the
  `downloads/*.pgml` and `downloads/*.pg` copies); and `problem_set_titles.yml`
  entries keyed by a bbq basename.
- SOURCE: `bbq-*-questions.txt` (the truth, always left in place); and
  topic-level `*.pgml` / `*.pg` sister masters (quarantined when orphaned, never
  deleted).

Why `downloads/` pgml/pg are generated but topic-level pgml/pg are masters:

- The `downloads/*.pgml` and `downloads/*.pg` files are reproducible copies.
  [run_bbq_tasks.py](../run_bbq_tasks.py) either regenerates them from YAML via
  `run_pgml_generation` (output dir `{topic}/downloads`, running
  `python3 <pgml_script> -y <yaml> -o ...`) or copies the sister file from the
  external generator-script directory via `copy_sister_pgml` (source
  `os.path.dirname(script_path)`, dest `{output_dir}/downloads`). The true
  masters live in the external `biology-problems/problems` repo, so the
  `downloads/` copies are always regenerable and orphans there are deleted.
- Topic-level `site_docs/<topic>/<x>.pgml` / `<x>.pg` are written by no known
  pipeline step (both pgml output paths target `downloads/`; `generate_pages`
  only searches via `find_pgml_file`). By user decision they are treated as
  masters and their orphans are quarantined (moved, never deleted) so an
  orphaned master surfaces in the flat `orphaned/` folder as
  `orphaned/<basename>` (no `<subject>/<topic>/` nesting) for human review
  instead of disappearing silently. A basename collision across topics
  hard-fails so a human can triage it.

File-class action table (the canonical positive rule each reconcile helper
implements; a live file keeps its current state in every class):

| File class | Orphan action |
| --- | --- |
| `downloads/selftest-*.html` | delete (git rm / os.remove) |
| `downloads/human_readable-*.html` | delete (git rm / os.remove) |
| `downloads/blackboard_qti_v2_1-*.zip`, `downloads/canvas_qti_v1_2-*.zip` | delete |
| `downloads/*.pgml`, `downloads/*.pg` | delete (reproducible copy) |
| `index.md` `selftest-<core>` include line | strip that one line |
| `problem_set_titles.yml` bbq-basename key | drop key (keep `last edit`) |
| topic-level `.pgml`, `.pg` | quarantine: git mv to `orphaned/<basename>` (flat) |
| `bbq-*-questions.txt` | source of truth: leave in place |
| coreless / unknown file | classify UNMANAGED: report only, no mutation |

The `orphaned/` quarantine folder lives at the repo root, outside
`docs_dir: site_docs`, so MkDocs never renders it.

## Where to add new work

- New site content: [site_docs/](../site_docs/).
- New subject or topic: [topics_metadata.yml](../topics_metadata.yml) plus a nav label
  in [mkdocs.yml](../mkdocs.yml).
- New generation logic: a module under [bioproblems_site/](../bioproblems_site/).
- New BBQ tasks: a per-subject CSV in
  [bbq_control/task_files/](../bbq_control/task_files/).
- New helper scripts: [tools/](../tools/).
- New docs: [docs/](.).
- New tests: [tests/](../tests/).
